# 校区海报展示

## 目标与边界

目标是把海报素材可靠地放进 16:9 展示画面，而不是重新设计海报：

- 一张海报一页，完整显示，不拉伸、不裁掉前景海报。
- 两侧使用同一海报生成的模糊暗背景，减少竖图在横屏上的空洞感。
- 每页默认停留 10 秒，淡入淡出，使用计时自动前进。
- 循环设置写入 `ppt/presProps.xml`，生成后自动检查关键 OOXML。
- 输出只有 PPTX；本技能不生成 HTML、MP4，也不负责发布到数字标牌平台。

开始前先确认：

1. 素材可用于本次校区展示；不含未授权学生姓名、正脸、学校、成绩单等敏感信息。
2. 输入目录第一层只放本次要展示的图片；脚本不会递归扫描子目录。
3. 超长海报即使完整放入一页，也可能因字号过小而不适合远距离观看。长宽比小于约 `0.55` 时，应优先拆页，而不是依赖自动缩小。

## 环境检查

技能脚本位于 `scripts/generate_pptx.py`。

优先使用当前 Codex 工作区自带的 Python；运行前验证依赖：

```bash
python3 -c "import pptx, PIL, lxml; print(pptx.__version__, PIL.__version__, lxml.__version__)"
```

最低要求：Python 3.10、`python-pptx>=1.0`、`Pillow>=10`、`lxml>=4`。仅在依赖确实缺失时安装：

```bash
python3 -m pip install "python-pptx>=1.0" "Pillow>=10" "lxml>=4"
```

## 标准流程

### 1. 收集并审查素材

在用户指定目录第一层查找 `.jpg`、`.jpeg`、`.png`，扩展名不区分大小写。先列出文件、检查尺寸与素材授权；不要把整张联络表、超长课程大纲或带透明黑底的拼图误当成适合大屏的单张海报。

### 2. 生成 PPTX

```bash
python3 "<技能目录>/scripts/generate_pptx.py" \
  --input "<海报目录>" \
  --output "<输出目录>/校区海报循环展示.pptx" \
  --seconds 10
```

参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--input` | 当前目录 | 只读取目录第一层的 JPG/JPEG/PNG |
| `--output` | `<输入目录>/artifacts/poster-display.pptx` | 输出 PPTX 路径 |
| `--seconds` | `10` | 每页停留秒数，必须大于 0 |

脚本会：

1. 按自然文件名顺序读取图片。
2. 应用 EXIF 方向；透明像素铺白。
3. 生成 1920×1080 合成画面，前景完整等比缩放，背景模糊压暗。
4. 生成 16:9 PPTX，写入 fade、自动前进、禁用点击前进、使用计时和循环播放设置。
5. 重新打开 PPTX 包，检查页数、比例、每页计时、fade 与循环设置；自检失败时返回非零退出码。

### 3. 验收

脚本输出“`OOXML 自检：通过`”只证明文件结构正确，不证明目标播放器一定遵循这些设置。交付前还必须：

- 渲染全部页面，逐页检查前景海报是否完整、清晰、无遮挡、无意外黑底。
- 对超长海报做远距离可读性审查；无法一眼读懂时拆页或更换素材。
- 在实际使用的 PowerPoint/WPS 和目标校区一体机上全屏试播至少一轮，确认 10 秒切换、淡入淡出、末页回到首页。
- 核对屏幕分辨率、缩放模式和播放设备。若不同播放器对 PPT 计时支持不一致，优先从 PowerPoint/WPS 导出 MP4，再用稳定播放器全屏循环。

## 常见失败

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| `第一层没有 JPG/JPEG/PNG` | 图片在子目录或格式不支持 | 把本次素材集中到一个目录第一层，或先转成 JPG/PNG |
| `处理图片失败` | 图片损坏、伪装扩展名或 Pillow 无法解码 | 重新导出该图片并单独验证 |
| PPTX 能打开但不自动翻页 | 播放器未使用计时、未进入放映模式或兼容性差 | 在目标播放器检查“使用计时”，全屏试播；必要时导出 MP4 |
| 海报很窄、文字读不清 | 原图过长，完整等比缩放后宽度太小 | 由设计/教研人员拆成多张 3:4 或接近竖版标准比例的页面 |
| 透明 PNG 出现黑块 | 黑色可能是原图中的不透明像素，不是透明区 | 回到源文件检查 alpha 通道并重新导出 |
